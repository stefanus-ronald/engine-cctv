📋 Apa itu LightNVR?

NVR (Network Video Recorder) open-source ditulis dalam C, dioptimalkan untuk perangkat hemat memori (awalnya Ingenic A1 SoC, RAM 256MB) tapi jalan di Linux apa saja.

🎯 Smart Detection

Detection zone berbasis poligon, integrasi object detection (ONNX/TFLite/SOD), ONVIF motion events, recording berbasis deteksi.

AI / CV

📺 Streaming Modern

WebRTC latency sub-detik + HLS untuk kompatibilitas, dapat di-toggle. Overlay deteksi real-time dengan bounding box.

go2rtc

🪶 Sangat Ringan

Jalan di SBC / SoC dengan RAM 256MB. Staggered init, JPEG-based detection, stream prioritization, swap support.

Embedded

🔐 Keamanan

Session token, TOTP/MFA (RFC 6238), role-based access (admin/user/viewer), API key, password hashing mbedTLS.

Auth

💾 Storage Cerdas

Retention berlapis 4 tier, recording protected, disk space management otomatis, retention beda untuk deteksi vs terjadwal.

Retention

🔌 Integrasi

MQTT untuk Home Assistant, ONVIF auto-discovery + PTZ, REST API 60+ endpoint, reverse proxy + HTTPS ready.

Open API

🧱 Arsitektur Inti

Insight terpenting: LightNVR tidak menangani koneksi kamera sendiri — semua I/O kamera didelegasikan ke go2rtc (proses Go terpisah yang di-spawn & dimonitor).

Kamera (RTSP/ONVIF) │ ▼ ┌─────────┐ WebRTC ──▶ Browser (live, latency sub-detik) │ go2rtc │ HLS ──▶ LightNVR HLS Writer ──▶ segmen .ts │ :1984 │ RTSP ──▶ LightNVR MP4 Recorder ──▶ file .mp4 └─────────┘ frame.jpeg ──▶ Detection (JPEG snapshot, hemat memori) │ ▼ SQLite (metadata: streams, recordings, detections, zones, users) │ ▼ Storage: /var/lib/lightnvr/data (recordings/mp4, recordings/hls, db)
🔑 Pola registrasi multi-source: setiap stream didaftarkan ke go2rtc dengan 3 source sekaligus — URL kamera asli, ffmpeg transcode audio→OPUS (WebRTC butuh OPUS bukan AAC), dan ffmpeg transcode video→H.264 (browser tak support WebRTC H.265).

🤖 Subsistem Detection AI

Satu unified detection thread per stream dengan state machine, mendukung 3 backend yang bisa di-switch per kamera.

🔹 SOD (Embedded)

Linked langsung ke binary. RealNet (face, ringan) / CNN (PASCAL VOC 20 class). Cocok untuk device kecil/offline.

On-device

🔹 TFLite / LiteRT

In-process, footprint sedang. Conditional compile -DHAVE_LITERT dengan delegate XNNPACK.

In-process

🔹 HTTP API

light-object-detect eksternal. POST JPEG → terima JSON. Memisahkan beban AI dari core → paling scalable.

Scalable
💡 Kunci hemat memori: detection mengambil JPEG snapshot dari go2rtc (/api/frame.jpeg?src=X&cache=30s) — bukan decode full video. Param cache 30 detik bikin tetap dapat frame walau kamera sempat disconnect.
State machine detection thread: INITIALIZING → CONNECTING → BUFFERING → RECORDING → POST_BUFFER → (kembali BUFFERING) │ ▲ │ deteksi │ flush pre-buffer ke MP4 └────────────┘ (dapat footage SEBELUM event)

⚖️ Engine CCTV Kita vs LightNVR

Kabar baik: ENGINE-CCTV kita sudah memakai backbone yang sama (go2rtc + FFmpeg). Jadi banyak pola LightNVR bisa diadopsi tanpa ganti fondasi. Perbedaannya ada di fokus & cakupan fitur.

🟢 ENGINE-CCTV (kita) — Node.js · MIT · zero-framework. Fokus Hikvision ISAPI: live WebRTC/MJPEG, playback dari NVR, deteksi hardware kamera (line-crossing, intrusion, face), NVR auto-sync semua channel, baca status HDD/SD. Tidak rekam sendiri — mengandalkan storage NVR.

💪 Kekuatan kita (pertahankan)

  • ISAPI mendalam — line-crossing arah panah, sensitivity, capabilities probe (LightNVR tak punya)
  • NVR auto-sync — import semua channel otomatis dari recorder
  • Playback dari NVR — tidak perlu rekam ulang, hemat storage besar
  • MIT + Node.js — bebas komersial, jauh lebih mudah dikembangkan/diaudit

🎯 Yang bisa kita serap dari LightNVR

  • AI object detection on-frame — person/car/dll, bukan cuma deteksi hardware kamera
  • Zona deteksi poligon — kurangi false positive
  • Pre-detection buffer + recording independen (opsional)
  • MQTT → Home Assistant, SQLite metadata, auth hardening

🛠️ Roadmap Implementasi ke Engine Kita

Urutan adopsi berdasarkan nilai vs effort. Semua hanya meniru pola/ide (legal) — bukan menyalin kode GPLv3. Karena kita sudah punya go2rtc + vca-proxy.js, beberapa fitur tinggal dirangkai.

① AI Object Detection on-frame PRIORITAS TINGGI · effort rendah

Saat ini deteksi kita 100% mengandalkan hardware kamera (ISAPI). Tambahkan deteksi AI server-side (person/car/dll) — fondasinya sudah ada semua di engine kita.

Pola LightNVR (terbukti hemat resource): go2rtc /api/frame.jpeg?src=ID&cache=30s ──▶ JPEG (tanpa decode video penuh) │ ▼ POST ke layanan AI Python ──▶ JSON {label, confidence, bbox} │ (kita SUDAH punya vca-proxy.js!) ▼ SSE broadcaster ──▶ overlay bounding box di tile (kita SUDAH punya SSE!)

Cara di engine kita: buat modul src/detection/ai-detection.js — interval per kamera ambil snapshot dari go2rtc (kita sudah jalankan go2rtc), kirim ke vca/vca-proxy.js (Python AI), hasilnya lewatkan ke events/sse-broadcaster.js yang sudah dipakai untuk alert ISAPI. Overlay di public/js/app.js sudah ada untuk ISAPI — tinggal pakai ulang.

② Zona Deteksi Poligon PRIORITAS TINGGI · effort sedang

Filter deteksi per-area agar tidak ada false positive (mis. hanya "person" di pintu, abaikan jalan raya).

Cara di engine kita: simpan poligon (koordinat normalized 0–1) per kamera di cameras.json field detection.zones[]. Filter pakai algoritma ray-casting point-in-polygon (LightNVR zone_filter.c) di modul AI detection. Editor poligon: tambahkan canvas overlay di settings modal public/js/app.js (kita sudah punya modal settings). Berlaku untuk AI detection maupun memfilter alert ISAPI.

③ MQTT → Home Assistant PRIORITAS SEDANG · effort rendah

Cara di engine kita: tambah dependency mqtt (npm), modul src/integrations/mqtt-client.js. Publish event deteksi (ISAPI + AI) ke broker dengan topic cctv/{camera}/detection + HA auto-discovery + motion state timeout 30s (pola LightNVR mqtt_client.c). Memungkinkan otomasi rumah/kantor dari event CCTV.

④ Recording Independen + Retention 4-Tier OPSIONAL · effort tinggi

Hanya jika butuh rekaman tidak bergantung NVR (mis. kamera IP tanpa NVR, atau backup kritikal). Untuk mayoritas kasus, andalkan NVR tetap lebih hemat.

Cara di engine kita: rekam dari RTSP go2rtc (rtsp://localhost:8554/ID) via FFmpeg -c copy ke MP4 tersegmen. Adopsi pre-detection buffer (circular buffer, rekam ~5 dtk sebelum event) dan retention 4-tier (Critical 3× / Important 2× / Standard 1× / Ephemeral 0.25× + protected). Butuh SQLite untuk index recording.

⑤ Hardening: SQLite, Auth, Health Monitor SAAT SCALE NAIK

  • SQLite — ganti JSON files saat data deteksi/recording membesar (query & histori lebih kuat). Pola migrasi versioned LightNVR.
  • Auth — TOTP/MFA + RBAC (admin/user/viewer), rate-limit login. Saat ini token opsional saja.
  • Health monitor consensus — restart go2rtc berdasarkan ">50% stream gagal", cooldown 120s (lebih pintar dari health check sederhana).
📌 Ringkas: Karena engine kita sudah berbagi fondasi go2rtc + punya SSE + punya vca-proxy ke Python AI, item ① & ② bisa dikerjakan cepat dan memberi lompatan fitur terbesar (deteksi AI cerdas + zona) tanpa mengubah arsitektur. Item ④ hanya kalau strategi storage berubah dari "andalkan NVR" ke "rekam sendiri".

💡 Pelajaran untuk Engine CCTV Kita

✅ Layak Diadopsi

  • Pola "go2rtc sebagai backbone" — jangan reinvent RTSP/WebRTC/ICE, ini keputusan arsitektur terbaik mereka
  • Detection via HTTP API eksternal — pisahkan beban AI dari core, mudah scaling/ganti model
  • JPEG snapshot untuk detection — bukan decode video, hemat memori & CPU drastis
  • Skema DB + retention 4 tier — blueprint matang, bisa jadi referensi tabel kita
  • Shutdown coordinator — shutdown berurutan by-priority, atomic + condition variable

⚠️ Perlu Diwaspadai

  • Lisensi GPLv3 — aman untuk belajar pola, TIDAK aman copy-paste kode ke produk proprietary tanpa commercial license
  • Codebase C low-level — manual thread/mutex/FFmpeg, maintenance berat
  • Fokus embedded 256MB — beberapa optimasi kurang relevan kalau target kita server besar / banyak kamera
  • Docker volume gotcha — jangan mount /var/lib/lightnvr langsung (menimpa web assets)

⚖️ Catatan Lisensi (PENTING)

GPLv3 + opsi Commercial License (OpenSensor Engineering).

✅ Aman: mempelajari arsitektur, pola desain, dan ide implementasi untuk engine kita.
❌ Tidak aman: menyalin/menurunkan kode LightNVR ke produk proprietary/tertutup tanpa membeli commercial license.

Rekomendasi: kalau engine CCTV kita mau closed-source, replikasi ide & pola-nya — jangan kode-nya. go2rtc sendiri berlisensi MIT (lebih permisif) sehingga aman dipakai sebagai backbone.
